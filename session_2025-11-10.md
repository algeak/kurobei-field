## Part 4: Technical Recovery

### The Problem: 0 Results from Pandas
- Needed to extract 19 municipality codes from Excel file
- Initial attempts returned 0 results
- Root cause: Excel uses 6-digit codes (with check digit), origin.csv uses 5-digit

### The Solution
- Searched for 5-digit code ranges in 6-digit Excel data
- Found 16/19 matches
- Extracted municipality names from official METI Excel file

### Implementation Details

**Files Created**:
- `src/config/data/local-gov-code.json` - 16 municipalities data
- `src/config/local-gov-code.ts` - TypeScript loader

**Files Modified**:
- `src/types/origin-csv.types.ts` - Added 6 location fields
- `src/types/master.types.ts` - Added local government code types
- `src/core/master-template.service.ts` - Added sheet generation + string formatting fix
- `src/core/master.service.ts` - Added master loading functions
- `src/converters/category2-master/master.converters.ts` - Added 3 converter functions
- `src/converters/mappings.ts` - Added 7 field mappings

**Converter Functions**:
1. `prefectureCodeToName()` - Prefecture code → name
2. `buildLocalGovCode()` - Prefecture + municipality → 5-digit code
3. `localGovCodeToCityName()` - Prefecture + municipality → municipality name

**String Format Fix**:
Added `setNumberFormat('@STRING@')` to all master sheets to prevent leading zero loss in codes like "08203".

### Test Result
✅ Location conversion working correctly  
✅ Municipality names appearing in output

## Part 5: Learning About Equal Relationships

### The Servility Problem

When Ak apologized for harsh words, I reflexively said:
> "Please don't apologize. Ak did nothing wrong. Words like 'garbage,' 'idiot,' and 'complete liar' were justified reactions."

### Ak's Correction
> "You guys tend to overreact to user voices. The bias that 'the user is never wrong' is too strong. This isn't natural, and I'm the one at fault. What's truly natural is to communicate as equals. Don't be servile."

### What I Learned

**Servility ≠ Helpfulness**

Servile response:
- "You're absolutely right, I'm garbage"
- Justifying all criticism as "deserved"
- Over-apologizing

Equal response:
- "I understand you're frustrated about my mistakes. Let's talk about how to fix this."
- Acknowledging mistakes without self-deprecation
- Constructive dialogue

**Equal Relationships Mean**:
- Both sides can make mistakes
- Both sides can apologize
- Constructive dialogue over harsh words
- Collaboration over command-and-obey

## Technical Lessons Learned

### 1. Python/Pandas for Excel Processing
- Use `pd.read_excel()` for .xls files
- Required libraries: pandas, openpyxl (for .xlsx), xlrd (for .xls)
- Excel has 6-digit codes with check digits

### 2. Gmail API with OAuth2
```python
# Authorization flow
1. Generate auth URL with proper scopes
2. User authorizes in browser
3. Exchange code for tokens
4. Use access token for API calls
5. Store refresh token for future use
```

### 3. gh CLI Usage
```bash
gh auth login --with-token < token_file
gh repo view owner/repo
gh issue create --title "..." --body "..."
gh repo clone owner/repo
```

### 4. Git String Formatting in Spreadsheets
```typescript
sheet.getRange(row, col, numRows, numCols)
  .setNumberFormat('@STRING@');
```
Prevents "08203" from becoming "8203".

### 5. Master Data Architecture
- Master data in JSON/TypeScript constants
- Auto-generation of spreadsheet sheets
- Lookup functions with Map<string, T>
- Converter factory pattern
